# 2026-03-05 问题与解决方案

## 问题描述

在教师端启动应用并进入考试列表页面时，
当浏览器/窗口的整体宽度在大约 **790px – 810px** 区间时，
页面出现持续抖动。

调查发现抖动在不断计算表格可用高度 (useTableHeight) 与
分页组件换行之间循环触发：

- 宽度临界时分页需要换行显示 `showSizeChanger` 等控件，
  导致分页高度增加。
- `useTableHeight` 监听到容器高度变化，重新计算表格 `scroll.y`，
  重新渲染时出现横向滚动条或移除滚动条。
- 横向滚动条状态变化又改变可用宽度触发分页换行，
  形成持续的高度震荡，从而看起来像“页面抖动”。

该问题在宽度大于或小于该区间时不存在，因为换行与滚动条
不再来回切换。


## 解决方案

为了打破抖动环，我们采取了以下措施：

1. **分页容器禁止换行**
   - 给分页外层 `div` 添加 `overflow-x-auto`，允许横向滚动而不是换行。
   - 为 `Pagination` 组件添 `className="xs-pagination-nowrap"` ，并
     在全局 CSS 中定义 `.xs-pagination-nowrap { flex-wrap: nowrap !important; white-space: nowrap; }`。
2. **保持高度稳定**
   - 由于分页始终单行显示，高度不再随着窗口宽度变化而抖动，
     `useTableHeight` 计算结果稳定，页面不再出现抖动。

该改动简单且兼容 Ant Design 与 Tailwind，用于所有类似情况。


### 补充建议

- 若需要进一步优化，可在窄屏时根据 `window.innerWidth` 关闭
  `showQuickJumper` 等可选控件以节省空间。
- 日后若引入更多分页定制，确保分页区域不会因内容尺寸变化
  导致父容器高度跳动。
