# 考试系统需求研究

## 目标

开发一款桌面端在线考试系统，面向局域网环境，可在网络不稳定时稳定运行。系统功能包括教师端创建与发布考试、导入题目并设置分值；学生端参加考试、查看答题进度与自动录屏；自动评分、成绩统计与报告。教师端支持分布式部署，当某台主机发生故障时，能够自动切换到其他主机继续服务。

## 调研发现

- 市面已有多种在线考试系统（如 Moodle、ExamSoft 等），但大多为基于 Web 的解决方案，缺乏适用于局域网的桌面端应用。
- 教师和学生对系统的易用性和稳定性要求较高，考试过程中不能出现卡顿或崩溃。
- 在网络状况不佳的环境下，局域网桌面端应用可以提供更佳的性能和用户体验。

## 核心需求

1. **教师端功能**

   - **创建考试**：可以设置考试名称、时间、题目类型（选择题、填空题、简答题等）和分值。
   - **导入题目**：支持从 Excel 或 CSV 文件导入题目，自动解析题目内容及分值。
   - **分布式部署**：支持多台教师主机部署，某台主机故障时可自动切换至其他主机继续运行。
   - **考试状态管理**：可发卷、开始、结束考试（支持手动或自动切换），实时查看考试进度、学生答题情况及学生端状态。

2. **学生端功能**

   - **参加考试**：学生登录后参加考试，系统显示考试时间和答题进度。
   - **自动录屏**：考试过程中自动录屏，记录答题过程以防作弊。
   - **无感重连**：网络断开后，学生仍可继续答题，恢复连接后自动同步状态。

3. **自动评分与成绩统计**
   - **自动评分**：选择题和填空题自动评分，简答题由教师手动评分。
   - **成绩统计**：统计考试成绩并生成报告，供教师和学生查看。
