# 2026-03-08 问题与解决方案

## 问题描述

执行 `git push origin` 时出现如下错误：

```text
fatal: unable to access 'https://github.com/XiaoSong-dada/xs-examination.git/': Recv failure: Connection was reset
```

无法把本地提交推送到 GitHub，报错提示连接被重置。

## 环境信息

- 系统：Windows
- 代理工具：Clash Verge
- 仓库路径：`E:\code\xs-examination`
- 远程地址：`https://github.com/XiaoSong-dada/xs-examination.git`

## 根因结论

- 本机直连 `github.com:443` 的 TCP 链路被重置，其它 HTTPS 站点正常。
- 问题并非 Git 自身配置或仓库内容，而是网络访问受限。
- 代理工具 Clash Verge 运行中，但最初使用错误的端口（`33331` 为管理接口），因此 Git 无代理时直连失败。
- Clash 配置的可用 HTTP/HTTPS 代理端口为 `mixed-port: 7897`，配置后访问 GitHub 恢复正常。

## 详细排查过程（可复用）

1. 检查远程地址与 Git 代理设置：
   ```bash
   git -C E:\code\xs-examination remote -v
   git -C E:\code\xs-examination config --show-origin --get-regexp "http\..*proxy|https\..*proxy"
   ```

2. 查看环境变量代理：
   ```powershell
   Write-Host "HTTP_PROXY=$env:HTTP_PROXY"
   Write-Host "HTTPS_PROXY=$env:HTTPS_PROXY"
   Write-Host "ALL_PROXY=$env:ALL_PROXY"
   ```

3. 测试 GitHub HTTPS 端口连通性：
   ```powershell
   Test-NetConnection github.com -Port 443 | Select-Object ComputerName,RemoteAddress,RemotePort,TcpTestSucceeded
   ```

4. 对比其他站点以确定是否是全局网络问题：
   ```powershell
   Test-NetConnection www.microsoft.com -Port 443 | Select-Object ComputerName,RemoteAddress,RemotePort,TcpTestSucceeded
   Test-NetConnection gitee.com -Port 443 | Select-Object ComputerName,RemoteAddress,RemotePort,TcpTestSucceeded
   ```

5. 测试备用通道 SSH-over-443：
   ```powershell
   Test-NetConnection ssh.github.com -Port 443 | Select-Object ComputerName,RemoteAddress,RemotePort,TcpTestSucceeded
   ```

6. 查找 Clash Verge 进程及其配置文件，确认代理端口：
   - Clash 配置文件位于 `%APPDATA%\io.github.clash-verge-rev.clash-verge-rev\clash-verge.yaml` 等，里面显示 `mixed-port: 7897`、`socks-port: 7898`。
   - 验证端口监听：
     ```powershell
     Test-NetConnection 127.0.0.1 -Port 7897
     ```

7. 临时通过该端口测试 GitHub 访问：
   ```bash
   git -C E:\code\xs-examination -c http.proxy=http://127.0.0.1:7897 \
      -c https.proxy=http://127.0.0.1:7897 ls-remote origin
   ```

## 解决方案

在仓库级别设置 Git HTTP/HTTPS 代理使用 Clash Verge `mixed-port`：

```bash
git -C E:\code\xs-examination config http.proxy http://127.0.0.1:7897
git -C E:\code\xs-examination config https.proxy http://127.0.0.1:7897
```

确认远程能访问：

```bash
git -C E:\code\xs-examination ls-remote origin
```

重新推送：

```bash
git -C E:\code\xs-examination push origin
```

推送成功，`main -> main`。

## 回滚或取消代理（如不再需要）

```bash
git -C E:\code\xs-examination config --unset http.proxy
git -C E:\code\xs-examination config --unset https.proxy
```

## 经验总结

- Clash Verge 的系统代理端口不一定与管理接口相同，必须看配置文件中的 `mixed-port`、`http-port`。
- `CONNECT tunnel failed, response 404` 表示连到了非代理服务端口。
- 遇到 GitHub 连接重置时优先检测本机到 `github.com:443` 的 TCP 连接。
